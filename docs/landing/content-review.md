# Сверка лендинга с `TZ.md`

## Что сверяем

- Запрещённые слова из раздела 2.1 `TZ.md`.
- Красные зоны из раздела 2.2 `TZ.md`.
- Обязательные дисклеймеры из раздела 2.3 `TZ.md`.

## Источник контента

Приоритетный источник — `web/content/landing-content.json`. Если JSON отсутствует, скрипт автоматически читает видимый текст из HTML-страниц каталога `web/` (кроме `/legal/*` и `404.html`).

## Как проверить

```bash
python scripts/check_landing_content.py
```

Чек подтверждает:
- отсутствие запрещённых слов (с учётом `allowListedPhrases` для обязательной фразы с «прогнозом») при наличии policy;
- отсутствие слов из красных зон при наличии policy;
- наличие всех обязательных дисклеймеров (в JSON-режиме — из `policy`, в HTML fallback — из базового набора).

## Результат в рамках текущей итерации

- Добавлены отдельные блоки до CTA в бот:
  - «Не является консультацией/прогнозом»;
  - «Возвратов нет».
- Дисклеймеры сохранены отдельным разделом на лендинге и продублированы перед оплатой в боте.
