## Правила работы с картой

1. Для каждой новой статьи сначала фиксируйте основной ключ и кластер в этой таблице.
2. Перед созданием нового URL проверяйте, что в таблице нет страницы с тем же интентом.
3. Если интент совпадает, обновляйте существующую страницу, а не создавайте конкурирующий URL.
4. Высокочастотные ключи закрепляйте за отдельными посадочными: одна страница = один основной интент.
5. Для смешанных сценариев (например, «число судьбы + предназначение») используйте отдельный URL только если это зафиксировано отдельной строкой в карте.

## Шаблоны meta description для кластеров

> Целевой коридор: **130–170 символов**. Формула: **запрос + выгода + online/Telegram CTA**.

1. **Кластер «расчёт / онлайн-калькулятор»**
   - Шаблон: `Нумерология онлайн расчёт по дате рождения: [ключевая выгода для пользователя]. Получите [формат результата] и продолжите разбор онлайн в Telegram.`
   - Пример: `Нумерология онлайн расчёт по дате рождения: быстро поймите сильные стороны и зоны роста. Получите структурированный отчёт и продолжите разбор онлайн в Telegram.`

2. **Кластеры «предназначение» и «деньги»**
   - Шаблон: `[Предназначение/финансовый код] по дате рождения: [практическая выгода]. Читайте разбор, берите конкретные шаги и переходите к персональному формату в Telegram.`
   - Пример: `Финансовый код по дате рождения: увидьте рабочие денежные сценарии и точки роста дохода. Читайте разбор, берите конкретные шаги и переходите к персональному формату в Telegram.`

3. **Кластер «ИИ-наталка»**
   - Шаблон: `ИИ натальная карта онлайн: [что пользователь получает без воды]. Сравните интерпретации, проверьте выводы на практике и запустите персональный разбор в Telegram.`
   - Пример: `ИИ натальная карта онлайн: понятная интерпретация сильных сторон, рисков и вектора развития. Сравните выводы на практике и запустите персональный разбор в Telegram.`

## Контроль длины мета-тегов

- `title`: держим в диапазоне **55–65 символов**.
- `description`: держим в диапазоне **130–170 символов**.
- Не повторяем один и тот же ключ более 2 раз в пределах одного meta-тега.
- Перед публикацией проверяем длину через локальный скрипт/one-liner (`len(title)`, `len(description)`).
