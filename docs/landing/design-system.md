# Landing Design System (one-screen)

## 1. Принципы

- Поддерживать атмосферу «современной саги» без визуального перегруза.
- Сохранять читаемость и контраст текста на любых фонах.
- Все решения ориентированы на one-screen лендинг: быстрый рендер, ясные акценты, один главный CTA.

## 2. Дизайн-токены

### 2.1 Цвета

#### Базовая палитра

- `--color-bg-base: #0B1020` — основной фон.
- `--color-bg-elevated: #131A2E` — карточки и поднятые поверхности.
- `--color-surface-glass: rgba(25, 34, 58, 0.72)` — стеклянные подложки.
- `--color-text-primary: #F5F7FF` — основной текст.
- `--color-text-secondary: #C3C9E6` — вторичный текст.
- `--color-text-muted: #98A2C7` — подписи и вспомогательные элементы.

#### Акцентные цвета

- `--color-accent-fire: #FF7A3D` — огненный акцент (CTA, highlight).
- `--color-accent-ice: #63C6FF` — ледяной акцент (secondary action, ссылки).
- `--color-accent-rune: #B28DFF` — руны/магические акценты.
- `--color-success: #4BD4A3` — успешные состояния.
- `--color-warning: #FFC65A` — предупреждения.
- `--color-danger: #FF6B7A` — ошибки.

#### Градиенты

- `--gradient-hero: linear-gradient(135deg, #121A31 0%, #1D2746 45%, #2A1F3F 100%)`.
- `--gradient-cta: linear-gradient(135deg, #FF7A3D 0%, #B28DFF 100%)`.
- `--gradient-metal: linear-gradient(150deg, #616A85 0%, #A4ADC9 48%, #4A526B 100%)`.

### 2.6 Glow/Metal эффекты

- Glow применять точечно: вокруг ключевого CTA, рунных иконок и hero-иллюстрации.
- Базовый glow: `0 0 0 1px rgba(178, 141, 255, 0.35), 0 0 24px rgba(178, 141, 255, 0.28)`.
- Fire glow (hover CTA): `0 0 28px rgba(255, 122, 61, 0.42)`.
- Ice glow (secondary): `0 0 22px rgba(99, 198, 255, 0.35)`.
- Metal-рамки для карточек: градиент `--gradient-metal` + прозрачная подложка, насыщенность не выше 70%.

## 3. Контрастность и доступность

- Минимальный порог WCAG 2.1:
  - Обычный текст: **не ниже 4.5:1**.
  - Крупный текст (от 24px или 19px bold): **не ниже 3:1**.
  - Границы интерактивных элементов и иконки: **не ниже 3:1** с фоном.
- Текст поверх сложных фонов (туман/руны/иллюстрации) размещать только через затемняющий overlay (`rgba(8, 12, 24, 0.55+)`).
- Не передавать состояние только цветом: добавлять изменение яркости, обводки, иконки или подписи.
- Фокус-стили обязательны для клавиатурной навигации.

## 4. Состояния кнопок

Для `CTA-button` и вторичных кнопок фиксируются 4 обязательных состояния:

- **normal**:
  - фон: `--gradient-cta` (primary) или `--color-bg-elevated` (secondary);
  - текст: `--color-text-primary`;
  - тень: `--shadow-cta` (primary) / `--shadow-soft` (secondary).
- **hover**:
  - яркость +6–8%;
  - лёгкий подъём (`transform: translateY(-1px)`);
  - усиленный glow (fire или ice в зависимости от типа).
- **focus**:
  - `outline: 2px solid --color-accent-ice`;
  - `outline-offset: 2px`;
  - без удаления системного focus ring.
- **disabled**:
  - фон приглушённый (`opacity` визуально эквивалентно 45–55%);
  - курсор `not-allowed`;
  - отключённые тени и glow;
  - контраст текста сохранять не ниже 3:1.

## 5. Стиль иллюстраций и фонов

Стилистика: «туман + руны + металл + огонь/лёд», но в **умеренной интенсивности**, чтобы не мешать чтению.

- **Туман**: мягкие полупрозрачные слои, без плотного «молока»; opacity обычно 8–18%.
- **Руны**: редкие световые символы, размытые по краям; использовать как акценты, не как паттерн на весь экран.
- **Металл**: холодные рамки/детали в карточках, сатиновый блеск без зеркальных бликов.
- **Огонь/лёд**: парные акценты для визуального баланса (тепло vs холод), без кислотных оттенков.
- Для мобильных устройств уменьшать интенсивность фона на 20–30%.

## 6. UI-компоненты (фиксированный набор)

### 6.1 `hero-card`

- Состав: заголовок, подзаголовок, главный CTA, короткий supporting text.
- Визуал: `--gradient-hero`, `--radius-xl`, `--shadow-card`.
- Ограничение: не более 2 строк подзаголовка на мобильных.

### 6.2 `pricing-card`

- Состав: название тарифа, цена, список преимуществ, кнопка выбора.
- Визуал: `--color-bg-elevated`, тонкая metal-рамка, акцент на рекомендованном тарифе через glow.
- Ограничение: одинаковая высота карточек в desktop-сетке.

### 6.3 `CTA-button`

- Состав: текст + опциональная иконка Telegram.
- Визуал: pill-радиус (`--radius-pill`), `--gradient-cta`, обязательные состояния из раздела 4.
- Ограничение: минимальная высота 48px, горизонтальные отступы не меньше `--space-6`.

### 6.4 `disclaimer-block`

- Состав: обязательные дисклеймеры из ТЗ.
- Визуал: спокойный контрастный контейнер с `--color-surface-glass`, без glow.
- Ограничение: максимально высокая читаемость, `font-size` не ниже 14px.

### 6.5 `FAQ-item`

- Состав: вопрос (trigger), ответ (collapsible body).
- Визуал: минималистичный аккордеон, аккуратная руна-иконка раскрытия.
- Ограничение: анимация раскрытия до 180ms, без «пружин» и резких эффектов.

## 7. Ограничения по производительности

- Избегать тяжёлых фоновых видео; приоритет — статичные/анимированные оптимизированные ассеты (WebP/AVIF/SVG/Lottie).
- Любые декоративные эффекты (blur, glow, parallax) использовать дозированно и только в ключевых местах.
- Суммарный вес критичных визуальных ассетов первого экрана — целиться в минимально возможный объём (в идеале до ~1–1.5 MB на мобильной сети).
- Загружать изображения через responsive-версии (`srcset`) и lazy-load для некритичного контента.
- Учитывать `prefers-reduced-motion`: отключать интенсивные анимации и свечение.
- На слабых устройствах использовать упрощённый режим (без сложных многослойных blur/particle-эффектов).
