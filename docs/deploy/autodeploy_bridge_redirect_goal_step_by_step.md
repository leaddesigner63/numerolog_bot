# Автодеплой bridge redirect + цель Метрики (`bridge_redirect`)

## Когда использовать

Используйте этот runbook при релизе изменений редирект-страниц:

- `web/ig/index.html`
- `web/vk/index.html`
- `web/yt/index.html`

Изменения добавляют событие `reachGoal` для цели `bridge_redirect` и передают `source` + `start_payload` в params.

## Шаги автодеплоя

1. Убедитесь, что изменения закоммичены в `main`/release-ветку.
2. Запустите GitHub Actions workflow `deploy.yml` (автодеплой).
3. Дождитесь успешного завершения джоба деплоя.
4. На сервере выполните smoke-check лендинга:
   ```bash
   ./scripts/smoke_check_landing.sh
   ```
5. Проверьте, что bridge-страницы открываются:
   - `/ig/`
   - `/vk/`
   - `/yt/`

## Настройка цели в Яндекс.Метрике

1. Откройте счётчик `106884182`.
2. Перейдите: **Настройки → Цели → Добавить цель**.
3. Тип цели: **JavaScript-событие**.
4. Идентификатор цели: `bridge_redirect`.
5. Сохраните цель.

## Проверка сбора после деплоя

1. Откройте в браузере:
   - `https://<домен>/ig/`
   - `https://<домен>/vk/`
   - `https://<домен>/yt/`
2. В Метрике откройте **Отчёты → События / Параметры визитов**.
3. Отфильтруйте событие `bridge_redirect`.
4. Проверьте разрезы по параметрам:
   - `source`: `ig`, `vk`, `yt`
   - `payload`: `ig_reels_1`, `vk_clips_1`, `yt_shorts_1`
   - `start_payload`: в params хита на соответствующей странице

## Откат

Если цель не собирается или редирект ломается:

1. Откатите коммит с изменениями bridge-страниц.
2. Перезапустите автодеплой через `deploy.yml`.
3. Повторите smoke-check `./scripts/smoke_check_landing.sh`.
